Matrix IDX setup checklist

The site is already live with the approved Matrix IDX URL. This checklist is only for creating cleaner dedicated Matrix configs later: Map Search, Full Search, and My Listings.

1. Create the Matrix configs

  1. Log into elevateMLS / PPMLS Matrix.
  2. Open the user menu in the upper-right.
  3. Go to Settings → IDX Configuration.
  4. Create or confirm these forms, in this order:
    • Map Search — primary embedded buyer search.
    • Search — full filter/search experience.
    • My Listings — Bibek/office/agent listing view if available.
  5. Turn IDX Enabled on for each one.
  6. If Matrix asks for allowed domains, use https://rimyan.com and https://www.rimyan.com.
  7. Copy only the generated Activation HTML/iFrame URL. Do not paste passwords, MLS credentials, or API keys anywhere in GitHub.

2. Paste the generated URLs into the site

Open script.js and replace these connector values:

  • idxMapSearch → Matrix Map Search URL.
  • idxAdvancedSearch → Matrix Search URL.
  • idxMyListings → Matrix My Listings URL.

The site already falls back safely: Map Search and Full Search use the current approved IDX URL; My Listings routes to the contact form until a dedicated My Listings URL exists.

3. Verify before merging

  • Map Search embeds in the MLS section.
  • Full Search opens Matrix in a new tab.
  • My Listings either opens Matrix or pre-fills the contact form.
  • Phone, email, brokerage, license, Equal Housing, and IDX attribution remain visible.
  • No MLS/API credentials are committed.

Do not scrape MLS pages or rebuild listing cards from iframe content. Keep listing data inside approved Matrix/elevateMLS surfaces.
